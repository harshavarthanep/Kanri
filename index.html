One-time setup: connect Google

Because Kanri has no server, it talks to Google directly from your browser. That needs a free OAuth Client ID from Google Cloud (no billing, ~10 minutes, once per team):

  1. Go to console.cloud.google.com → create a project (any name).
  2. Open APIs & Services → Library and enable Google Drive API and Google Sheets API.
  3. Open APIs & Services → OAuth consent screen → External → fill app name & your email. Add your teammates' emails under Test users (or publish the app later).
  4. Open Credentials → Create credentials → OAuth client ID → Web application. Under Authorized JavaScript origins add your GitHub Pages URL, e.g. https://yourname.github.io (and http://localhost:8000 for testing).
  5. Copy the Client ID (ends in .apps.googleusercontent.com) and paste it below.

The ID is saved in this browser and embedded in your page for teammates — it is a public identifier, not a secret. While the consent screen is in “Testing” mode, only listed test users can sign in.
